# OrderSystem - Микросервисная система обработки заказов

## 📋 Описание проекта

**OrderSystem** — это распределенная система для обработки заказов, построенная на микросервисной архитектуре. Система демонстрирует использование современных технологий и паттернов для построения отказоустойчивых и масштабируемых приложений.

### 🎯 Основные возможности

- ✅ Создание заказов с выбором товаров из каталога
- ✅ Автоматическая проверка наличия товаров на складе
- ✅ Обработка платежей (симуляция)
- ✅ Управление складскими запасами
- ✅ Отказоустойчивость через компенсирующие транзакции (Saga pattern)
- ✅ Асинхронное взаимодействие через Kafka

### 🏗️ Общая архитектура

[Client] → [API Gateway] → [Order Service]
↓ Kafka → [Stock Service] → [Payment Service] → [Notification Service]

## Поток данных

### Успешный сценарий заказа

1. POST /orders (Customer → API Gateway → Order Service)
2. Order Service сохраняет заказ → публикует OrderCreated в Kafka
3. Stock Service читает OrderCreated → резервирует товары → публикует StockReserved
4. Payment Service читает StockReserved → обрабатывает оплату → публикует PaymentCompleted
5. Order Service читает PaymentCompleted → обновляет статус заказа → публикует OrderCompleted
6. Notification Service читает OrderCompleted → отправляет уведомление

### Сценарий с ошибкой (компенсация)

1. OrderCreated → StockReserved (успешно)
2. PaymentFailed (ошибка оплаты)
3. Stock Service читает PaymentFailed → освобождает зарезервированные товары
4. Order Service читает PaymentFailed → помечает заказ как Failed
5. Notification Service отправляет уведомление об ошибке
